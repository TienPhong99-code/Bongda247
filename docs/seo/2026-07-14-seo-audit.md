# SEO Audit — Bongda247 (2026-07-14)

Tổng hợp từ: khảo sát theme/site local + agent SEO (schema, content) + kiến thức technical WordPress. Site: WP classic theme `bongda247`, local `http://bongda247.local`, production dự kiến `bongda247.blog`.

## Hiện trạng (tóm tắt)

- Theme **trắng** meta SEO: không OG/Twitter cards, không meta description, không JSON-LD, không breadcrumb. Chỉ `language_attributes()` + `wp_head()`.
- **Chưa cài** plugin SEO (RankMath/Yoast). Có `robots.txt` + `wp-sitemap.xml` (WP core).
- `<html lang="en-US">` — **Site Language WP đang để English** (nội dung toàn tiếng Việt) → cần đổi.
- Bài seed hiện ~90 từ/bài → **rủi ro "scaled content abuse"** (chính sách spam Google 9/2024 về nội dung AI hàng loạt).
- CPT `match_insight`: `public=true`, `has_archive=false`, slug `nhan-dinh-tran` → bài lẻ truy cập được nhưng **không có trang hub "Nhận định"**.
- Pages đã có: Bảng xếp hạng, Lịch thi đấu, Giới thiệu, Liên hệ, Chính sách bảo mật, Điều khoản. Rác: "Sample Page" (xoá), "Privacy Policy" draft trùng.

## (B) YÊU CẦU SEO

### B1. Nền tảng meta + schema (Cao)
- **OG + Twitter cards** mọi trang (title, description, image, type article).
- **Meta description** động (excerpt bài / mô tả category).
- **JSON-LD** (mẫu chi tiết đã có từ agent schema):
  - `Organization` + `WebSite` + `SearchAction` (site-wide, `wp_head` priority 1) — sitelinks searchbox.
  - `NewsArticle` + `BreadcrumbList` (single.php) — Top Stories/Discover + breadcrumb SERP.
  - `SportsEvent` (match_insight) — entity signal; map `match_date`→startDate, teams→homeTeam/awayTeam.
  - `CollectionPage`/`ItemList` (archive/lưới) — hỗ trợ crawl/entity.
- **Lưu ý NewsArticle:** nội dung AI-generated — dùng `NewsArticle` vẫn index tốt nhưng KHÔNG lên Top Stories carousel tới khi được duyệt Google News Publisher. Không bị phạt.

### B2. Config nhanh (Cao)
- **Đổi Site Language → Tiếng Việt** (Settings → General) → `lang="vi"`.
- Xác nhận permalink `/%category%/%postname%/`.
- **`noindex` trang tìm kiếm** `/?s=` (tránh index trang mỏng).
- robots.txt trỏ sitemap; kiểm `wp-sitemap.xml` có gồm category + CPT.

### B3. Template (Trung)
- **Author box** cuối `single.php` (tên + link author + bio).
- **Ngày cập nhật** (`dateModified`) hiển thị khi khác ngày đăng + trong JSON-LD.
- **Breadcrumb** hiển thị (UI) khớp `BreadcrumbList` schema.
- **Bài viết liên quan** (3 bài cùng category) cuối `single.php`.
- **TOC** cho bài dài (parse H2, anchor link) — tốt cho AI Overview.
- **1 `<h1>`/trang**, heading hierarchy đúng; `alt`/`figcaption` ảnh cụ thể (không lặp tiêu đề).

### B4. Nội dung / quy trình bot (Cao — chống thin content)
- Prompt bot tăng **độ dài tối thiểu**: tin ≥600 từ, nhận định ≥1000 từ; **validation từ chối** bài <500 từ (báo Telegram).
- **Alt text cụ thể** khi upload media (`lib/wp.js`) thay vì dùng tiêu đề bài.
- **Lead sentence + số liệu cụ thể** mỗi section (AI-citation): "Arsenal đứng thứ 2, 71 điểm, bất bại 12 trận" — dữ liệu BXH đã có từ football-data.
- Render `source_credit`/`source_url` rõ (đã lưu meta, cần hiển thị đẹp).

### B5. Chọn hướng triển khai meta/schema — QUYẾT ĐỊNH CHÍNH
- **RankMath (plugin):** tự lo OG + meta description + Article/NewsArticle + Breadcrumb + sitemap + redirect — nhanh, phủ rộng. Nhược: thêm dependency; **PHẢI tắt schema tự viết nếu có** (tránh trùng publisher trong GSC).
- **Hand-code trong theme:** toàn quyền, không plugin, khớp phong cách custom hiện tại (đã tự viết football-data). Nhược: tự bảo trì OG/schema/sitemap.
- **Không chạy cả hai** cho cùng một loại schema.

## Lộ trình đề xuất (tác động cao/công sức thấp trước)
1. Config: Site Language → vi; noindex search. (phút)
2. Chọn RankMath **hoặc** hand-code → OG + meta + JSON-LD (Organization/WebSite + NewsArticle + Breadcrumb).
3. Template: author box + ngày cập nhật + bài liên quan + breadcrumb UI.
4. Page mới: hub **Nhận định** (`has_archive`), **Kết quả**, mở rộng Giới thiệu + trang Chính sách biên tập.
5. Bot: tăng độ dài + validation chống thin content; alt text; lead-sentence số liệu.
6. Phase 2: trang đội bóng, cẩm nang evergreen, llms.txt.
